1. Keitaro — база

Keitaro — трекер. Сидит между рекламой (Facebook) и сайтом казино. Его задача — понять, откуда пришёл игрок, по какому объявлению кликнул и сколько ты заработал.
Facebook → Keitaro → Казино (оффер)

Основные понятия

ТерминОписание
Кампания (Campaign)Один эксперимент. Одна кампания = одна связка (креатив + оффер + аудитория). Создаёшь → получаешь ссылку для Facebook.
Потоки (Streams)Несколько запасных направлений внутри одной кампании. Keitaro распределяет трафик по потокам в процентах.
Лендинги (Landings)Страница до казино (пре-лендинг). Не обязательна.
Офферы (Offers)Целевая страница — сайт казино, куда ведёшь игрока.
Правила (Rules)Фильтры: страна, устройство, время, IP. Можно перенаправлять разные сегменты на разные офферы.
PostbackОбратный сигнал: казино сообщает Keitaro, что клик привёл к конверсии.
Click IDУникальный идентификатор каждого клика. «Номер заказа», связывающий клик и конверсию.

Правила (Rules) — подробнее

Самая мощная фича Keitaro:

  • Фильтрация трафика: показывать только iOS, только Германию, только с 18:00 до 23:00
  • Перенаправление: iPhone → один оффер, Android → другой
  • Чёрные списки: заблокировать IP-адреса скликеров

Потоки (Streams) — как работают

Пример для казино:

  • Поток 1: оффер казино A (платит $100 за депозит)
  • Поток 2: оффер казино B (платит $80, но берёт больше гео)
  • Поток 3: заглушка («скоро открытие», если другие не взяли)

Распределение: 50% / 30% / 20% — настраивается вручную.

Колонки в статистике Keitaro

КолонкаЧто значит
HitsВсе переходы (клики)
ClicksУникальные клики (без ботов и повторов)
ConversionsЦелевые действия (регистрации / депозиты)
CR (%)Conversion Rate = Конверсии / Клики × 100
RevenueСколько заработал
CostСколько потратил на рекламу
ProfitRevenue − Cost
ROI (%)Profit / Cost × 100
EPCRevenue / Clicks (средний доход с клика)
CPACost / Conversions (стоимость одного действия)

Ошибки новичков

❌ Льют трафик без трекера → ты слепой.
❌ Не настраивают postback → Keitaro не видит конверсий.
❌ Сложные правила с первого дня → сначала один поток и один оффер.

2. Полная схема работы

Facebook → Клоака → Keitaro → Преленд → PWA / Казино
CPA-сеть ← Postback → Keitaro
ШагЧто происходит
1Байер создаёт кампанию в Facebook, вставляет ссылку из Keitaro
2Игрок кликает → попадает в Keitaro (или сначала в клоаку)
3Keitaro проверяет правила (страна, устройство, время)
4Перенаправляет на сайт казино / PWA
5Игрок регистрируется / вносит депозит
6Казино отправляет postback в Keitaro
7Keitaro записывает: клик → конверсия → профит
8Ты видишь статистику

Что передаётся в URL при клике

Facebook автоматически добавляет параметры в ссылку:

https://трекер.com/click/123?
  fbclid={fbclid}&         ← уникальный ID клика от Facebook
  country={country}&       ← страна пользователя
  device={device}&          ← устройство
  os={os}&                  ← ОС
  creative={creative_id}&   ← ID креатива
  campaign={campaign_name}& ← название кампании
  adset={adset_name}&       ← группа объявлений
  ad={ad_name}               ← объявление

Keitaro всё это парсит — ты видишь, какой креатив принёс больше конверсий.

3. Креативы (Creatives)

Креатив — это объявление, которое видит пользователь в Facebook. Картинка, видео, текст, заголовок — всё вместе.

Из чего состоит креатив в Facebook

ЭлементОписаниеПример для казино
МедиаКартинка или видеоВидео: «Джекпот сорвал 10 секунд назад»
Основной текстТекст над медиа«Только сегодня — 200% на первый депозит»
ЗаголовокЖирный текст под медиа«Casino XYZ — фриспины без депозита»
КнопкаПризыв к действию«Забрать бонус» / «Играть»
ОписаниеКороткий текст«Лицензия Кюрасао, вывод за 5 минут»
DestinationКуда ведёт ссылкаСсылка из Keitaro

Виды креативов для казино

ТипОписание
ИмиджевыеБренд, элитность, красивые интерьеры. Дорого, статусно.
БонусныеВ центре — халява: «100 фриспинов без депозита», «+300%».
Вин-креативыВыигрыш: скриншот баланса, видео джекпота.
Новостные / нативныеМаскируются под новость: «Врач выиграл $50,000 играя в слоты».
ТрэшовыеНизкокачественные картинки с жирными стрелками. Часто работают лучше красивых.

Как байеры работают с креативами

  • Спай-сервисы: AdHeart, BigSpy — подсматривают креативы конкурентов
  • Связка креативов: 3-5 разных креативов на одну аудиторию
  • Ротация: креатив «выжигается» за 2-5 дней, надо заливать новый

Что аналитик контролирует

  • CTR по креативам — какой лучше
  • CR по креативам — какой лучше конвертит
  • Сколько живёт креатив (на 3-й день CTR падает?)

4. PWA (Progressive Web App)

PWA — веб-сайт, который выглядит и ведёт себя как мобильное приложение. Не требует установки из App Store или Google Play.

Зачем PWA в гемблинге

Apple (App Store) и Google (Play Market) запрещают приложения казино.
Facebook запрещает рекламу приложений казино в магазинах.

Решение — PWA:

  • Не нужно загружать в магазины приложений
  • Facebook не может проверить, что внутри
  • Байер рекламирует «браузерную игру» или «приложение для развлечения»

PWA vs Webview

PWAWebview (обёртка)
Что этоСайт с возможностью установкиРеальное приложение, внутри — сайт
Где живётНа главном экранеВ списке приложений
Нужен магазинНетДа (APK файл)
МодерацияНетДа (Apple/Google проверяют)
Ограничения FacebookНетДа (удаляют приложение)

Технические особенности

  • После установки выглядит как родное приложение
  • Работает без адресной строки браузера
  • Быстрее за счёт кэширования (Service Worker)
  • Может отправлять push-уведомления

Проблемы PWA

  • iOS: PWA работает хуже — Safari ограничивает Service Worker, push-уведомлений нормальных нет
  • Криво сделанный PWA может не передавать Click ID → теряешь конверсии
  • PWA нужно обновлять вручную при смене дизайна казино

Что проверять, если используется PWA

  • Установки vs регистрации: ставят PWA, но не регистрируются — проблема на первом экране
  • Время клик → конверсия: если PWA грузится долго — конверсий меньше
  • Устройства: на iOS может работать хуже, чем на Android

5. Техническая сторона связки Facebook → Keitaro

Что происходит в момент клика

  1. Браузер пользователя → переходит по ссылке из объявления
  2. Запрос долетает до сервера Keitaro
  3. Сервер проверяет: кампания активна? поток работает? правила?
  4. Делается редирект на целевой URL
  5. Всё записывается в базу: клик, источник, юзер-агент, IP, страна, устройство

Всё занимает 10-200 мс.

Типы редиректов

ТипОписаниеПлюсыМинусы
302 (HTTP)Сервер отвечает «иди туда»Просто, быстроFacebook видит конечный URL
Meta RefreshПоказывается «Подождите...»Можно вставить задержкуДольше, хуже UX
JavaScriptРедирект через JSСкрывает реферерБлокируется на iOS
IframeОффер внутри страницы трекераПользователь не покидает доменМедленно, проблемы с куками

Домены и их роль

Самая больная тема. Facebook банит домены, которые ведут на казино. Байеру нужно постоянно покупать новые.
  • Покупают пачками в зонах .com, .xyz, .top, .click, .online (~$1-2)
  • Домен живёт от нескольких часов до нескольких недель
Пользователь видит: vk-news24.com (преленд) → Keitaro: tracker.domain.com → casinosite.com (оффер)

6. Преленд (Pre-lander)

Преленд — страница, которую пользователь видит ДО того, как попадёт на сайт казино.

Зачем нужен

  • Прогрев: объяснить пользователю предложение
  • Фильтрация: отсеять ботов, нецелевых
  • Сбор данных: телефон, email (для повторных продаж)
  • Маскировка: скрыть от Facebook, что трафик идёт на казино

Типы прелендов

ТипКак выглядитДля чего
Квиз«Ответь на 3 вопроса и получи бонус»Сбор данных, вовлечение
Новостной«Игрок из [город] выиграл джекпот!»Доверие, нативность
Заглушка«Приложение скоро запустится»Сбор контактов
Демо-слотИгровой автомат прямо на страницеВовлечение сразу
PWA-установка«Установи через браузер»Установка PWA

Как преленд влияет на статистику

  • На каждом шаге теряется 10-30% пользователей
  • Если преленд не передаёт Click ID — конверсия не привяжется
  • Если преленд грузится > 3 секунд — половина пользователей уходит

8. Postback — техническая механика

Postback — HTTP-запрос от сервера казино / CPA-сети к Keitaro. Сигнал: «Этот клик привёл к конверсии, вот выплата».

Как выглядит postback-запрос

GET http://keitaro.domain.com/postback?
  clickid=ABC123&       ← ID клика из Keitaro
  status=approved&       ← статус конверсии
  payout=75&             ← выплата
  currency=USD&          ← валюта
  ip=1.2.3.4&            ← IP пользователя
  offer_id=123&          ← ID оффера
  action_id=456789       ← ID транзакции

Click ID — сердце системы

Когда пользователь кликает, Keitaro создаёт запись:
clickid = "5f8a3b2c1d9e4f7a", IP = 1.2.3.4, timestamp = 2026-07-08 14:30:00, campaign_id = 42

Когда приходит postback с тем же clickid — Keitaro находит запись и дописывает конверсию.

Как проверить, что postback работает

Шаг 1: Проверь, передаётся ли Click ID в URL

Открой кампанию и посмотри финальный URL:

https://casino.com/?clickid={clickid}&sub1={campaign_id}

Перейди по ссылке сам — виден ли clickid в адресной строке?

Шаг 2: Логи Keitaro

Keitaro → Reports → Logs

Отфильтруй по кампании. Ищи:

  • Hit = клик (должен быть)
  • Conversion / Postback = конверсия (должна прийти через некоторое время)

Шаг 3: Тестовый postback руками

Возьми реальный clickid из логов и отправь:

Invoke-WebRequest -Uri "http://tracker.domain.com/postback?
  clickid=5f8a3b2c1d9e4f7a&payout=100&status=approved"

200 OK → postback работает. Проблема на стороне CPA-сети.

Шаг 4: Проверь настройки CPA-сети

Макросы Click ID в разных сетях:

СетьМакрос
ClickDealer{clickid}
Leadbit{clickid}
AdCombo{sub1}
MundoMedia{aff_click_id}
Bittraffic{click_id}

Шаг 5: Сквозной тест

  1. Создай тестовую кампанию
  2. Перейди по ссылке с телефона
  3. Зарегистрируйся в казино
  4. Внеси минимальный депозит ($1-5)
  5. Подожди 10-30 минут
  6. Проверь логи Keitaro

Типичные проблемы с postback

ПроблемаПричинаРешение
Hit есть, Conversion нетPostback не пришёлПроверь настройки в CPA-сети
Conversion есть, Payout = 0Не передаётся суммаНастрой макрос {payout}
Conversion в Pending бесконечноКазино не подтверждаетПиши в поддержку сети
Click ID пустойKeitaro не назначаетПроверь настройки кампании

9. Facebook Pixel (Пиксель)

Пиксель — невидимый счётчик, который Facebook ставит на сайт. Когда кто-то заходит на сайт, пиксель говорит Facebook: «Этот пользователь у меня на сайте».

Зачем нужен

  • Считать конверсии — Facebook видит, кто кликнул и что сделал
  • Оптимизировать показы — показывает объявления тем, кто чаще конвертится
  • Ретаргетинг — собирает людей для повторных показов
  • Look-alike аудитории — находит похожих на конвертировавшихся

Как выглядит код пикселя

<script>
  !function(f,b,e,v,n,t,s){
    if(f.fbq)return;n=f.fbq=function(){...};
    fbq('init', '123456789012345');
    fbq('track', 'PageView');
  }</script>

Проблема: казино не ставят пиксель

Казино обычно отказывается ставить пиксель Facebook на свой сайт, потому что:
  • Facebook получит данные о посетителях
  • Facebook может заблокировать аккаунт казино

Как обходят в гемблинге

Вариант 1: Пиксель на преленде

Facebook → Преленд (тут пиксель) → Казино

Facebook считает конверсией действие на преленде, не видя казино.

Вариант 2: Пиксель через Keitaro

Keitaro умеет генерировать пиксель и вставлять его в цепочку редиректа. Настраивается в кампании.

Вариант 3: Conversions API (CAPI)

Сервер отправляет данные о конверсиях напрямую в Facebook через API. Не зависит от браузера, блокировщиков и iOS ограничений.

События пикселя

СобытиеКогда срабатывает
PageViewПри любом заходе на страницу
LeadПользователь оставил контакты
RegistrationПользователь создал аккаунт
PurchaseВнёс депозит

iOS 14+ — проблема для пикселя

iOS 14.5+ требует согласия на трекинг. ~50% отказываются. CAPI (серверные конверсии) теперь обязателен — без него Facebook видит только половину конверсий.

11. Ретаргетинг

Ретаргетинг — показ рекламы людям, которые уже взаимодействовали с твоей рекламой или сайтом.

Как работает

  1. Человек кликнул по объявлению → попал на преленд (там пиксель)
  2. Посмотрел, но не зарегистрировался
  3. Пиксель записал: «Вася был на сайте, не конвертился»
  4. Facebook показывает Васе другие объявления

Типы аудиторий для ретаргетинга

АудиторияКто тамЧто показывают
Все посетителиЛюбой, кто открыл прелендНовые креативы
Не конверсияБыл > 10 сек, не зарегился«А ты видел бонус?»
Брошенная корзинаНачал регистрацию, бросил«Добей регистрацию, получи фриспин»
Зарегился, нет депозитаЕсть аккаунт, нет денег«Внеси депозит → +100%»
Лучшие клиенты> 3 депозитовVIP-предложения

Окна ретаргетинга

ОкноКогда использовать
1-3 дняСвежие, дёшево вернуть
7-14 днейСтандартный ретаргетинг для гемблинга
30-180 днейДолгий ретаргетинг (брендовый)

Ретаргетинг + Keitaro (Rules by Cookie)

В Keitaro можно перенаправлять пользователя на другой поток при повторном клике:

  • 1-й клик: Поток A (стандартный оффер)
  • 2-й клик: Поток B (оффер с бонусом 200% вместо 100%)

⚠️ Что проверять как аналитик

Красный флаг: холодный трафик не конвертится, а ретаргетинг даёт все конверсии. Это значит, что байер забил на холодную аудиторию и живёт только за счёт прогретых. Масштабировать такую связку нельзя.

12. Фрод (Fraud)

Фрод — мошенничество, обман, нечестные действия. Искусственное завышение статистики или воровство бюджета.

Виды фрода

1. Клик-фрод (Click Fraud)

Фейковые клики, которых на самом деле не было.

  • Как делают: бот-фермы, сервисы скликивания, скрипты
  • Признаки: CTR > 5%, трафик по расписанию, один User-Agent, повторяющиеся IP

2. Конверсионный фрод (Conversion Fraud)

Фейковые регистрации и депозиты.

  • Как делают: мусорные email, виртуальные карты, друзья «зарегайся за подарок»
  • Признаки: все конверсии в одно время, одинаковые email-домены (@mailinator.com), Conversion Time = 0

3. Инсентив-трафик (Incentive Traffic)

Байер платит людям за действия.

  • Как делают: «зарегайся получи 10 рублей», форумы, группы
  • Чем плохо: конверсии отклоняются через 7-30 дней (Rejected), но байер уже получил выплату

4. Атрибуционный фрод (Attribution Fraud)

Кража чужих конверсий. Перехват postback и подмена Click ID.

5. Фрод по Postback

Байер сам шлёт фейковые postback-запросы в Keitaro.

  • Признаки: postback приходят быстрее, чем возможна реальная конверсия; одинаковое время у многих конверсий

6. Return / Chargeback Fraud

Байер знает, что часть конверсий откатят, и закладывает это в стратегию — успевает вывести деньги до пересчёта.

Инструменты Keitaro против фрода

ИнструментЧто делает

13. Клоакинг (Cloaking)

Клоакинг — технология, показывающая разный контент разным людям. Facebook-модератору — безопасную страницу, пользователю — казино.
Facebook (модератор) → Белый контент (безопасно)
Пользователь → Казино

Как клоака определяет, кто кликнул

Screen ResolutionУ модератора 1920×1080, у пользователя 375×812

Наличие Facebook PixelЕсть ли пиксель на странице?

Важно для аналитика

  • Клоака стоит ДО Keitaro. Keitaro видит только «хороший» трафик
  • Если клоака сломалась — Keitaro не получает трафик, конверсии падают до 0
  • Байер может отключить клоаку и лить напрямую → бан аккаунта
  • Клоака ≠ качество трафика. Если байер гонит ботов через клоаку — это всё ещё фрод

Клоакинг vs Фрод

Клоакинг — это не фрод для CPA (трафик реальный), но нарушение правил Facebook.
Фрод — обман CPA-сети. Клоакинг — обход модерации Facebook.

14. Полный цикл: от креатива до выплаты

Этап 1: Подготовка (1-3 дня)

  • Выбрать оффер в CPA-сети
  • Подготовить 3-5 креативов (спай-сервисы + дизайнер)
  • Настроить кампанию в Keitaro (поток, Sub-ID, ссылка)
  • Купить 5-10 доменов
  • Подготовить/прогреть аккаунты Facebook

Этап 2: Запуск (30 мин + модерация до 24 ч)

  • Создать кампанию в Facebook Ads Manager
  • Выбрать цель (Traffic / Conversions)
  • Настроить аудиторию (гео, пол, возраст)
  • Загрузить креатив, вставить ссылку Keitaro
  • Установить бюджет $50-100/день

Этап 3: Трафик пошёл

Первые 30-60 минут:

Facebook: 100 кликов, потрачено $20
Keitaro:  95 hits, 0 конверсий (пока)

Этап 4: Первая конверсия (15-60 мин после клика)

Игрок: клик → регистрация → депозит €50
Postback: clickid=ABC123 & status=approved & payout=100

Байер видит: потрачено $20, заработано $100, ROI = 400%

Этап 5: Холд (7-30 дней)

  • День 0-3: Pending (проверка качества)
  • День 3: Approved или Rejected
  • День 7-30: Hold (заморозка на случай возврата)
  • День 30+: выплата

Этап 6: Скейл

  • Увеличение бюджета: $50 → $200 → $500/день
  • Запуск похожих кампаний (новые креативы, новая аудитория, другие гео)
  • Клонирование успешных связок

Этап 7: Выплата (30-45 дней от конверсии)

ДеньСобытие
1 январяКонверсия
3 январяApproved
30 январяХолд закончился
5 февраляВыплата (USDT, PayPal, Wise)

Этап 8: Аналитика

Считает байер:

Доход от CPA:        $10,000
Facebook:              -$6,000
Инструменты (Keitaro и т.д.): -$669
Чистая прибыль:        $3,331

Считаешь ты (реальная прибыль):

Чистая прибыль:                $3,331
Rejected (20%):                 -$2,000
Chargeback (5%):                -$500
Реальная прибыль:               $831

Горячий vs Холодный ROI

ПериодROIЧто учтено
Горячий (Pending)+400%Все конверсии, включая сомнительные
Реальный (Approved через неделю)+150%Часть ушла в Rejected

Если реальный ROI значительно ниже горячего — либо байер гонит некачественный трафик, либо накручивает конверсии.
