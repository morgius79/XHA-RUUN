# RULE-0033: Классификация объектов реестра

**Тип:** канон
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Каждый объект реестра получает классификационный ярлык (tag) в секции «Тип» карточки, выбираемый из фиксированного набора категорий.

## Описание

Набор тегов: графемы — «consonant», «vowel», «diacritic», «punctuation»; лексемы — «root», «affix», «particle», «name», «term»; правила — «grammar», «naming», «canon», «style», «technical»; локации — «city», «region», «landmark», «water», «structure»; события — «canonical», «meta»; персонажи — «major», «minor», «deity», «mythical». Тег записывается в нижнем регистре. Один объект может иметь только один основной тег.

## Примеры

1. Соблюдение: L-000001 — «root»; RULE-0001 — «grammar»; LOC-0001 — «city»; CH-0001 — «major».
2. Нарушение: отсутствие тега; множественные теги; тег не из фиксированного набора; несоответствие тега содержанию объекта.
