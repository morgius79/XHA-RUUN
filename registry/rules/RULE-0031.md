# RULE-0031: Процесс изменения канона

**Тип:** канон
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Любое изменение канона (добавление, изменение, удаление объекта) проходит через процедуру: issue — proposal — review — merge — publish.

## Описание

Процесс обеспечивает целостность канона. Issue описывает необходимость изменения. Proposal содержит конкретные правки. Review включает проверку перекрёстных ссылок и целостности реестров. Merge применяет изменения в ветку main. Publish обновляет ROADMAP.md и RELEASE.md с указанием новой версии. Для критических исправлений (опечатки, ошибки в ID) процедура может быть сокращена до review — merge — publish.

## Примеры

1. Соблюдение: добавление новой лексемы L-000501: issue — proposal (карточка + связи) — review — merge в main — publish v0.1.1.
2. Нарушение: прямое изменение карточки в main без proposal и review; отсутствие обновления ROADMAP.md после изменения.
