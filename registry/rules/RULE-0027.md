# RULE-0027: Порядок регистрации объектов

**Тип:** канон
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Регистрация объектов канона (графем, лексем, правил, локаций, событий, персонажей) выполняется в порядке возрастания ID последовательно, без пропусков и пересортировки.

## Описание

Каждый новый объект получает следующий свободный ID в своей категории. Пропуски ID не допускаются — если объект удалён, его ID помечается как retired и не переиспользуется. Регистрация ведётся в соответствующих реестрах (registry/graphemes/, registry/lexemes/, registry/rules/, registry/locations/, registry/events/, registry/characters/). Объект считается зарегистрированным только после создания файла карточки.

## Примеры

1. Соблюдение: L-000001, L-000002, L-000003 — последовательно; при удалении L-000002 он становится retired, следующий — L-000004.
2. Нарушение: пропуск ID без пометки retired; присвоение ID вне очереди; повторное использование retired ID.
