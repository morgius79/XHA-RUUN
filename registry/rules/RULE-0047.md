# RULE-0047: Валидация карточек правил

**Тип:** технические
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Все карточки правил (RULE-*) проходят автоматическую валидацию: проверка структуры секций, формата ID, существования связанных правил, соответствия шаблону.

## Описание

Валидация выполняется скриптом build/validate-rules.{py,ps1} и включает: (1) наличие всех обязательных секций (Формулировка, Описание, Примеры, Связанные правила); (2) корректность ID (формат RULE-NNNN); (3) существование всех файлов, указанных в Связанных правилах; (4) соответствие статуса разрешённым значениям (active, retired). Результат валидации выводится в консоль и записывается в build/validation-report.log.

## Примеры

1. Соблюдение: прогон валидатора после создания RULE-0050 — все проверки пройдены, отчёт пуст.
2. Нарушение: RULE-0001 ссылается на RULE-0099, которого не существует; отсутствует секция Примеры; ID записан как RULE-01.

## Связанные правила

- RULE-0049: Проверка ссылок
