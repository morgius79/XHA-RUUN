# RULE-0049: Проверка целостности ссылок

**Тип:** технические
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Все перекрёстные ссылки между файлами проекта проверяются на целостность перед каждым релизом: каждый ID, упомянутый в тексте, должен соответствовать существующему файлу карточки.

## Описание

Проверка выполняется скриптом build/check-links.{py,ps1}, который: (1) сканирует все .md-файлы; (2) извлекает все вхождения паттернов [ID] и (ID); (3) сверяет их со списком файлов в registry/; (4) сообщает о битых ссылках. Игнорируются ссылки на retired объекты (допускаются). Результат записывается в build/link-check-report.log. Релиз не публикуется при наличии битых ссылок.

## Примеры

1. Соблюдение: скрипт не находит нарушений — все ссылки валидны, отчёт пуст.
2. Нарушение: ссылка на L-999999 (не существует); ссылка на RULE-0001 нарушена, так как файл перемещён в _retired/ без обновления ссылок.
