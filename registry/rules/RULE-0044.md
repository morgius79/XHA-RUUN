# RULE-0044: Именование файлов контента

**Тип:** стиль
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Файлы контента именуются по шаблону [категория]-[короткое_описание].md, где категория определяет префикс, а описание — kebab-case на русском или английском.

## Описание

Категории: encyclopedia/ — без префикса; registry/graphemes/ — G-; registry/lexemes/ — L-; registry/rules/ — RULE-; registry/locations/ — LOC-; registry/events/ — EV-; registry/characters/ — CH-. Описание — до 5 слов в kebab-case. Допускается использование английского для технических файлов и русского для контентных. Файлы в корне проекта (README, ROADMAP, RELEASE, CLAUDE, PROJECT_CONTEXT) префиксов не имеют.

## Примеры

1. Соблюдение: «RULE-0001.md», «L-000001.md», «encyclopedia/glava-1-vozniknovenie.md».
2. Нарушение: файл карточки без ID в имени; пробелы в имени файла; русские буквы в техническом файле; несоответствие префикса категории.
