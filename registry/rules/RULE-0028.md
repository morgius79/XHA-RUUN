# RULE-0028: Уникальность ID

**Тип:** канон
**Статус:** active
**Создан:** 2026-07-16

## Формулировка

Каждый объект канона имеет уникальный ID в своей категории; ID-номера не повторяются и не переиспользуются после удаления объекта.

## Описание

ID состоит из префикса категории (G- / L- / RULE- / LOC- / EV- / CH-) и шестизначного (для лексем) или четырёхзначного (для остальных) порядкового номера. Уникальность гарантируется тем, что retired ID не возвращаются в пул. При конфликте (например, два объекта претендуют на один ID) приоритет имеет объект, зарегистрированный первым по времени.

## Примеры

1. Соблюдение: L-000001 и L-000002 — разные лексемы; L-000001 retired, следующий — L-000003.
2. Нарушение: создание L-000001 второй раз после удаления первого; дублирующиеся ID в одной категории.
