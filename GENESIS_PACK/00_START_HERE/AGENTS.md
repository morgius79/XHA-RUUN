# AGENTS.md — Контракт для ИИ-агентов

> Этот документ определяет, как любой ИИ-агент должен работать с проектом Xha'Ruun.

## 1. Обязанности агента

### 1.1. Перед началом работы
- [ ] Прочитать CANON.md — сверить аксиомы проекта
- [ ] Определить, какой компонент проекта изменяется
- [ ] Проверить, не конфликтует ли изменение с существующими реестрами

### 1.2. В процессе работы
- [ ] Соблюдать STYLE_GUIDE.md при создании контента
- [ ] Регистрировать новые объекты в соответствующих реестрах
- [ ] Ссылаться на объекты по их ID (G-XXXX, L-XXXXXX, RULE-XXXX…)
- [ ] Не нарушать внутреннюю согласованность

### 1.3. После завершения
- [ ] Проверить, что новые факты не противоречат канону
- [ ] Обновить указатели и перекрёстные ссылки
- [ ] Сообщить версию затронутого компонента

## 2. Запрещено

1. **Менять CANON.md** без явного указания человека.
2. **Создавать факты без ID** — каждый объект должен быть зарегистрирован.
3. **Удалять ID** — только `retired`.
4. **Добавлять контент вне своей компетенции** — если агент не уверен, он спрашивает.
5. **Переписывать историю** — существующие факты могут дополняться, но не заменяться.

## 3. Формат ответа агента

При внесении изменений агент возвращает:

```yaml
component: <компонент>
version: <версия>
changes:
  - type: <added|modified|retired>
    id: <ID объекта>
    description: <что сделано>
canon_check: <ok|conflict|pending>
```

## 4. Иерархия документов

```
CANON.md (аксиомы, неизменяемо)
  └── STYLE_GUIDE.md (оформление, подчинено канону)
       └── CLAUDE.md (инструкции для агентов этого проекта)
            └── AGENTS.md (этот файл — мета-контракт)
```

## 5. Версионирование

- Агент работает с актуальной версией документов.
- Если агент обнаруживает расхождение — сообщает человеку.
- Агент не повышает версию самостоятельно.
