# CLAUDE.md — Xha'Ruun Project Guide

> Инструкция для Claude Code при работе над проектом Xha'Ruun.

## Проект

Xha'Ruun — научно-фантастическая вселенная, цивилизация, язык и многотомная энциклопедия.  
Проект ведётся как **software-репозиторий**: с версионированием, CI, тестами и инструментами сборки.

## Структура репозитория

```
/                       # Корень проекта
├── GENESIS_PACK/       # Каркас, канон, документация
│   └── 00_START_HERE/  # Входная точка (этот файл)
├── xharuun/            # Энциклопедия (главы, сборка)
├── scripts/            # Инструменты сборки
├── assets/             # Ресурсы (изображения, карты)
├── tools/              # SDK и генераторы
├── registry/           # Реестры объектов
└── volumes/            # Черновики томов
```

## Как работать

### Принципы

1. **Не нарушать канон** — любое изменение проверять по CANON.md.
2. **Одна логическая единица за коммит**.
3. **ID объектов не переиспользовать** — удалённые объекты получают статус `retired`.
4. **Ссылаться через ID**, а не через названия.

### Формат коммитов

```
[том/реестр/инструмент] Краткое описание
```

Примеры:
- `[registry] Added G-0032 to G-0048`
- `[volume-1] Chapter 007 – Atmospheric composition`
- `[canon] Added Law of Silence clarification`

### Стиль Markdown

- Заголовки: ATX (`# ## ###`)
- Таблицы: выравнивание через `|---|`
- Код: обратные кавычки или ```блоки```
- Ссылки: относительные, через ID где возможно

## Сборка

```bash
npm run build:md        # Сборка энциклопедии в Markdown
npm run build:pdf:sha   # Сборка PDF
npm run validate        # Валидация ссылок
npm run build:font      # Генерация шрифта Sharuu
```

## Команды для Claude Code

- `claude` — запуск с OmniRoute (см. start.bat)
- Все операции — через обычные git-команды

## Важно

- CANON.md — **неизменяемые аксиомы**. Изменение только через повышение версии.
- Все новые факты должны быть согласованы с существующим каноном.
- Не добавлять факты "на вырост" — только то, что нужно сейчас.
